AIテスト自動化プラットフォーム 開発者向け仕様書 v3.0

この開発者向け仕様書は、AIテスト自動化プラットフォーム「QA³」の技術仕様、アーキテクチャ、実装計画を詳細に解説します。システムの品質保証を完全自動化し、開発チームの生産性を飛躍的に向上させる革新的ソリューションです。

🚀 デモを体験する 💼 ビジネス情報を見る

Service Overview

🎯 サービス概要

QA³は、AIが自律的にWebアプリケーションをテストし、バグを発見・報告する革新的プラットフォームです。

🤖

1. 自律的な品質検証

準備やセッティングは不要。AIが自ら「どこを検証すべきか」を学習し、自律的にテストを開始します。

  • • シナリオ性とランダム性の両立
  • • 機械学習による継続的な学習
  • • エッジケースの自動発見
📋

2. PMが見てもわかりやすいレポート

どのような検証をしたのか／不具合再現方法／スクリーンショット／緊急度／原因の予想を含むレポートを自動生成。

  • • 自動スクリーンショット撮影
  • • 詳細な再現手順
  • • AIによる重要度判定
🔗

3. シームレスなツール連携

CI/CD、AIコーディングツール（Cursorがメイン）と連携し、検知された不具合に対して自動でPRを作成。

  • • GitHub Actions / Jenkins連携
  • • Cursor自動PR作成
  • • 品質ゲート機能
🎯

4. 高精度な不具合発見

ソースコード（GitHubリポジトリ）や操作マニュアル／仕様書が提供されると、AIがより精度高くシナリオを生成・実行。

  • • ソースコード解析
  • • 仕様書理解
  • • 自動質問書発行

Phase 1

🔧 MVP: コア体験の提供

QA³の核となる価値「AIによるテスト自動化とバグ報告」を体験していただくための最小限の機能を実装します。単一ページでの「おまかせ」モードによる自律的テストが中心となります。

機能仕様

  • おまかせモード: LLMが画面のHTMLから意味を読み取り、挙動的な問題がないかを自律的にテスト
  • コンテキスト理解: サービス説明をLLMに提供することで、より適切なテスト戦略を自動生成
  • 環境変数管理: キー・バリュー形式で環境変数を管理、テスト実行時に{{変数名}}として参照可能
  • 単一ページ限定: Phase 1では1つのページ内での操作テストのみ（複数ページ遷移はPhase 2から）
  • バリデーション: URL形式チェック、必須項目の入力確認を実装
  • リアルタイム進行表示: テスト実行中のAIの思考プロセスと操作をリアルタイム可視化

テストアクション定義

AIエージェントが実行可能なアクションの種類と仕様

基本アクション

CLICK

指定した要素をクリック

例: CLICK "#login-button"
INPUT

テキストフィールドに文字を入力

例: INPUT "user@example.com" INTO "[name='email']"
SCROLL

ページをスクロール

例: SCROLL "down" 500px

高度なアクション

WAIT

指定した条件まで待機

例: WAIT FOR element-visible "#result"
ASSERT

期待値と実際の値を比較

例: ASSERT text "ログイン成功" IN ".message"
SCREENSHOT

スクリーンショットを撮影

例: SCREENSHOT "login-completed"

Test Modes

💡 テストモード詳細

QA³が提供する4つのテストモードの詳細仕様と使い分け方法を解説します。

🤖

1. おまかせモード

AIが画面を見ながら自律的に判断し、最適なテストシナリオを実行

動作原理

HTMLから画面の意味を理解 → コンテキスト分析 → 挙動的問題の検出 → 自律的テスト実行

適用場面

単一ページの機能検証、UI/UXの挙動テスト、初期品質チェック

実行時間

5-15分（ページ複雑さにより調整）

📋

2. シナリオモード

特定のシナリオに限定してテストを実施

シナリオ記法（複数ページ対応）
GOTO "{{BASE_URL}}/login"
INPUT "{{EMAIL}}" INTO "[name='email']"
CLICK "#submit-btn"
GOTO "{{BASE_URL}}/dashboard"
ASSERT text "ようこそ" IN ".welcome"
適用場面

複数ページのユーザージャーニー、リグレッションテスト、重要機能の定期検証

実行時間

5-20分（シナリオ複雑さ・ページ数に依存）

⚡

3. ハイブリッドモード

おまかせとシナリオを同時並行で実行

実行方式

シナリオ実行後に自律探索を継続

適用場面

包括的品質保証、重要リリース前テスト

実行時間

15-45分（最も網羅的）

🎯

4. 指示モード

特定の不具合再現チェックを自動実施

入力例

"ログイン後にダッシュボードが表示されない"

適用場面

お客様報告バグの一次切り分け

実行時間

5-15分（再現試行回数に依存）

Glossary

📚 用語集

QA³で使用される専門用語と概念の定義一覧です。

用語 定義 詳細・補足
シナリオ
Scenario
 特定の目的を達成するための一連の定義済み操作フロー テキストファイルや専用エディタで定義可能。例：「ユーザー登録」「商品購入」
テストセッション
Test Session
 あるテストの開始から終了までの一回の実行単位 セッションは特定のシナリオを持つ場合と持たない場合がある
テストアクション
Test Action
 テストセッション内でAIエージェントが実行する個別の操作イベント クリック、テキスト入力、スクロール、スワイプ、待機など
バグチケット
Bug Ticket
 検出された不具合に関する情報を集約したレポート 従来の「不具合カード」に代わる呼称
AIエージェント
AI Agent
 ブラウザを自律的に操作するAIシステム 機械学習モデルにより画面を解析し最適なアクションを決定
品質ゲート
Quality Gate
 CI/CDパイプラインで品質基準を満たさない場合に自動でデプロイを停止する仕組み 重要度High以上のバグが発見された場合にブロック
ビジュアルリグレッション
Visual Regression
 UIの見た目の変化を画像比較により自動検出するテスト手法 Phase 2で実装予定。ピクセル単位での差分検出が可能
HAR
HTTP Archive
 Webブラウザとサーバー間の通信ログを記録するファイル形式 ネットワークエラーやパフォーマンス問題の分析に使用
